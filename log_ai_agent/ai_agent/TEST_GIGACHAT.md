# Инструкция по запуску GigaChat ассистента

## Предварительные требования

1. Убедитесь, что установлены и запущены все необходимые Docker-контейнеры:
   ```bash
   docker ps
   ```
   Должны быть запущены контейнеры:
   - cyberlog-db (база данных PostgreSQL)
   - cyberlog-backend
   - cyberlog-frontend

2. Убедитесь, что файл `.env` существует в директории `log_ai_agent/` и содержит корректный `GIGACHAT_API_KEY`.

## Запуск GigaChat ассистента

1. Откройте терминал в директории проекта

2. Активируйте виртуальное окружение (если используется):
   ```bash
   # Для Windows
   .venv\Scripts\activate
   
   # Для Linux/Mac
   source .venv/bin/activate
   ```

3. Перейдите в директорию агента:
   ```bash
   cd log_ai_agent/ai_agent
   ```

4. Запустите скрипт gigachat_chat.py:
   ```bash
   python gigachat_chat.py
   ```

## Ожидаемый результат

После успешного запуска вы должны увидеть приветственное сообщение:
```
Привет! Я ассистент по кибербезопасности. Введите 'exit' для выхода.
```

## Возможные проблемы и решения

### 1. Ошибка подключения к базе данных
```
could not translate host name "db" to address
```
**Решение:** Убедитесь, что в файле `gigachat_chat.py` настройка хоста установлена на `localhost`:
```python
"host": "localhost",  # Direct connection from host to container
```

### 2. Ошибка аутентификации в GigaChat
```
Error communicating with GigaChat: (URL('https://ngw.devices.sberbank.ru:9443/api/v2/oauth'), 400, b'{"code":4,"message":"Can't decode \'Authorization\' header"}')
```
**Возможные причины и решения:**
- Проверьте срок действия токена `GIGACHAT_API_KEY`
- Убедитесь, что токен скопирован полностью и без лишних пробелов
- Проверьте, что у токена есть необходимые права доступа к API
- Убедитесь, что файл `.env` находится в правильной директории (`log_ai_agent/.env`)

## Команды в ассистенте

- Обычный текст: задайте любой вопрос по кибербезопасности
- `exit`: для выхода из ассистента

## Дополнительная информация

Ассистент сохраняет историю чата в PostgreSQL базе данных и использует RAG-контекст для анализа логов. История чата будет загружаться при каждом новом запуске.
