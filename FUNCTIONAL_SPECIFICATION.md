Это функциональное задание описывает новый функционал и доработку имеющегося функционала для существующей системы. Оно детализирует задачи, добавляя цели, критерии приемки и зависимости для явной реализации.

## Цели проекта
Система должна повысить точность анализа логов, обеспечить автономность, автоматизировать пайплайны отчетов и стать более удобной.

**Результат:** готовый продукт для production.

[Схема системы](https://ibb.co/zH6JQzyK)

## Задачи

### Высокий приоритет
- выбрать и установить локальную LLM на отдельный сервер*
- полностью переработать модуль с RAG (новый подход*)
- выстроить пайплайн от получения логов до готовых отчетов*
- доработать определение типа угрозы и уровня серьезности

### Средний приоритет
- доработать метрику эффективности качества (RAGAS)
- добавить модуль подключения ко внешнему источнику логов (Push API)
- добавить мониторинг всех этапов работы работы LLM (логи)
- написать полную документацию для работы с системой

### Низкий приоритет
- переработать веб-интерфейсы по новому дизайну
- добавить ведение логов работы сотрудников
- доработать уведомление пользователей о новых отчетах
- добавить модуль распознавания речи (speech salute bot)
- доработать презентацию проекта

### Нефункциональные требования
- Производительность: <10с на полный анализ лога
- Безопасность: шифрование логов
- Масштабируемость: Docker
- Тестирование: Unit-тесты (80% покрытия)

## Примечания

**Новый подход для определения угроз, аномалий:**
1. получение логов
2. передача логов LLM
3. запрос объяснения логов от LLM
4. сопоставление объяснения с MITRE (RAG)
5. формирование отчета на основе сопоставления с помощью LLM

**Использовать модели с квантизацией чисел.**

**Пайплайн работы системы:**
1. получение логов от внешнего источника (Push API)
2. первоначальная обработка логов
3. передача логов LLM для анализа + специальный промпт
4. получение анализа от LLM
5. сопоставление анализа с базой MITRE с помощью RAG
6. отправка всей полученной информации LLM + специальный промпт
7. определение уровня серьезности и типа угрозы
8. формирование отчета на основе ответа от LLM
9. сохранение отчета в базе данных
10. отправка уведомления о новом отчете пользователя

**!** на каждом этапе ведутся логи **!**
